# Restringir Acceso y Datos a Usuarios no Autenticados

Asegura que un usuario debe estar logueado para interactuar con cualquier función de la app. Si no está logueado, se lo redirecciona a la pantalla de login, no se le muestra ningún dato y el backend rechaza cualquier intento de leer o registrar información.

## User Review Required

> [!IMPORTANT]
> **Filtro de Datos a Nivel de Base de Datos:**
> Además de redireccionar al usuario en el frontend, filtraremos los gastos en el backend mediante el campo `usuarioId`. Esto garantiza que los usuarios logueados solo puedan ver y guardar sus propios datos, resolviendo la seguridad de punta a punta.

    *   Borrar el localStorage (`localStorage.clear()`).
    *   Intentar ingresar directamente a `http://localhost:5173/` o `http://localhost:5173/gastos`.
    *   Comprobar que redirige automáticamente a `http://localhost:5173/pages/login` y no muestra el layout del dashboard.
